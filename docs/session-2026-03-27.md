## What Was Built This Session

### bricklayer-cli (Bricks 18–28)

**Brick 18 — close-session docs sync**
Upgraded `bricklayer close-session` to write directly to `decision-log.md`
and `pipeline-status.md` at `DOCS_PATH` after every sprint review.
Uses `llama-3.3-70b-versatile` for structured extraction. Pipe sanitization
added to prevent markdown table corruption. Replaces the manual `!scribe`
Discord ritual for CLI sessions.

**Brick 19 — bricklayer new-project**
Scaffolds a new project at `context/projects/<n>/` with `STATE.md`,
`decision-log.md`, and `state.json`. Validates project name format.
Duplicate name check. `context/projects/` created if missing.

**Brick 20 — bricklayer context**
Prints a compact context block for a project — state, last decisions,
next command — ready to paste into any AI session. Reads from
`context/projects/<n>/` directory. `--project` flag or falls back
to current project in `state.json`.

**Brick 21 — Agent registry**
`context/agents/registry.yaml` — master index of all agents with ID,
project, role, runtime, status, trigger, location. Pre-populated with
3 live agents. `cli/registry.py` module with `load()`, `get()`, `add()`,
`update_status()` — atomic writes, schema validation.

**Brick 22 — bricklayer agent list + status**
`bricklayer agent list` prints formatted table of all agents.
`bricklayer agent status <id>` prints full detail block.
Empty registry → clear prompt to run `bricklayer agent new`.

**Brick 23 — bricklayer agent new**
Scaffolds a new agent directory from `nanobot-template` or raw Python
scaffold. Fills in all `__PLACEHOLDER__` values in template files.
Registers agent in `registry.yaml` with `status: stopped`.
Validates ID format: `[a-z]+-[a-z]+-[0-9]{2}`.
Template path read from `bricklayer.yaml agents.nanobot_template`.

**Brick 24 — bricklayer agent deploy**
Copies agent directory to `DEPLOY_REPO_PATH`, commits, pushes, prints
exact VPS docker commands. Idempotent — handles "nothing to commit"
gracefully. Delete-then-copy prevents ghost files in deploy repo.
`TimeoutExpired` and `FileNotFoundError` handled cleanly.
`bricklayer agent live` marks agent live after VPS confirms running.

**Brick 25 — Auto-.env loading + LLM config from bricklayer.yaml**
`bricklayer` auto-loads `.env` from project root at startup.
No manual `source .env` needed ever. LLM provider, model, heavy_model,
and api_key_env all read from `bricklayer.yaml llm:` section.
Switch from Groq to OpenAI/Anthropic by changing `bricklayer.yaml`
and `.env` — zero code changes. Non-overwrite rule: `.env` never
beats explicitly exported shell vars.

**Brick 26 — State hardening**
Two fixes: (1) `close-phase` and `close-feature` crashed with
`ValueError: current_brick must be str, got NoneType` — fixed by
coercing null to `""` before state write. (2) `state.json not found`
crashed every command on a fresh repo — fixed by auto-creating
`state.json` with defaults, including creating the `bricklayer/`
parent directory if missing.

**Brick 27 — close-session --summary flag**
Added required `--summary` flag to `bricklayer close-session`.
Without it, the 8b model wrote garbage entries like
"sprint review completed / skeptic_packet_ready" because it only
had `state.json` fields as context. Groq now receives the human-written
summary as primary input. State fields appended as context only.
Running without `--summary` → clear error, exit 1.

**Brick 28 — Auto-push docs after close-session**
After writing docs to `DOCS_PATH`, `bricklayer close-session` now
automatically runs `git add docs/ && git commit && git push` in the
`DOCS_PATH` directory. GitHub becomes the single source of truth
immediately after every session. Push failure is never fatal —
docs are written locally regardless.

## Architectural Decisions Locked

### Docs Are Per-Project (critical)
`DOCS_PATH` in each project's `.env` points to that project's own
`docs/` directory. Never share `DOCS_PATH` across projects.

```
idea-to-product-os: DOCS_PATH=/path/to/ai-agents/docs
reddit-monitor:     DOCS_PATH=/path/to/reddit-monitor/docs
client-project:     DOCS_PATH=/path/to/client-project/docs
```

### Session Scribe Discord is Fallback
`bricklayer close-session --summary "..."` is the primary docs writer.
`!scribe` in Discord is the fallback for mobile / away from terminal.

### Brick Handoffs Are Tool-Agnostic
```
📋 BRICK [N]: [Name]

Load AGENT.md. Follow Autonomous Build Mode.
[Claude Code: /run-brick | Copilot CLI / Codex: run the 7-step sequence]
```

### Three-Way Agent Runtime Routing (locked)
```
Raw Python  → trivial, stable, single-purpose, no tool-calling
NanoBot     → scheduled workers, 1-3 tools, MCP, simple memory
AgentScope  → Directors, 4+ tools, DB memory, HTTP endpoint
```

### Multi-Tool AI Strategy (verified)
Claude Code, GitHub Copilot CLI (Claude Sonnet 4.5), OpenAI Codex (GPT-5.4).
All interchangeable via `bricklayer pause` → new tool → `bricklayer resume`.

Tool entry points:
```
CLAUDE.md                        → Claude Code
AGENTS.md                        → Copilot CLI (reads automatically)
.github/copilot-instructions.md  → Copilot CLI (repo-level)
CODEX.md                         → OpenAI Codex
.cursorrules                     → Cursor
AGENT.md                         → source of truth for all tools
```

### LLM Provider Config in bricklayer.yaml
```yaml
llm:
  provider: groq
  model: llama-3.1-8b-instant
  heavy_model: llama-3.3-70b-versatile
  api_key_env: GROQ_API_KEY
```
Switch providers: change `bricklayer.yaml` + `.env`. Zero code changes.

## New Project Workflow

### Mental Model
```
idea-to-product-os  = the factory (tools, standards, pipeline)
my-new-project      = a product built IN the factory
ai-agents           = the warehouse (running agents, shared)

You never edit the factory to build a product.
You use the factory's tools in the product's workspace.
```

### Stage 1 — Planning (Claude.ai browser project)
```
Venture OS → stress-test idea → Org Schema
Agent-OS   → design agents → PROJECT BRIEF per agent
plan-brain → break into bricks → Brick 1 handoff
```

### Stage 2 — Project Setup (terminal, one time per project)
```bash
mkdir my-project && cd my-project
git init
git remote add origin git@github.com:hermon1738/my-project.git

cp /path/to/idea-to-product-os/templates/bricklayer.yaml .
cp /path/to/idea-to-product-os/templates/env.example .env

# Edit bricklayer.yaml — replace /path/to/idea-to-product-os with real path
# Edit .env:
#   GROQ_API_KEY=gsk_your_real_key
#   DOCS_PATH=/path/to/my-project/docs    ← this project only
#   DEPLOY_REPO_PATH=/path/to/ai-agents

mkdir docs
echo "# my-project — Decision Log" > docs/decision-log.md
echo "| Date | Component | Decision Made | Status | Next Action |" >> docs/decision-log.md
echo "| --- | --- | --- | --- | --- |" >> docs/decision-log.md
echo "# my-project — Pipeline Status" > docs/pipeline-status.md

git add . && git commit -m "init: my-project"
git push -u origin main

bricklayer status   # verify: works, no errors
```

### Stage 3 — Build (terminal + AI tool, per brick)
```bash
# Session start (every time):
bricklayer resume
bricklayer status
bricklayer next

# Open AI tool:
claude --dangerously-skip-permissions   # Claude Code
copilot                                  # GitHub Copilot CLI
codex                                    # OpenAI Codex

# Create branches:
bricklayer branch --feature my-feature
bricklayer branch --phase 1 first-phase
bricklayer branch 1 first-brick

# Paste brick handoff into AI tool
# AI runs Autonomous Build Mode (Steps 1–7)
# AI posts report → you send to Gemini → write verdict
# bricklayer build --verdict PASS → brick closed, merged to phase

# All bricks in phase done:
bricklayer close-phase
# → merged to feature, landed on feature

# Next phase from updated feature (includes all prior phase changes):
bricklayer branch --phase 2 second-phase

# All phases done:
bricklayer close-feature
# → merged to main
```

### Stage 4 — End of Every Session
```bash
bricklayer close-session --summary "what you built, what's next"
# → Groq sprint review (70b model)
# → writes session-log.md locally
# → writes docs/decision-log.md + pipeline-status.md at DOCS_PATH
# → git push to GitHub automatically
```

### Stage 5 — Tool Switching (context limit)
```bash
bricklayer pause
# → writes HANDOFF.json

# Open new tool, first message:
bricklayer resume
# → full context restored, continue exactly where you left off
```

### Stage 6 — Deploy an Agent
```bash
bricklayer agent new \
  --id my-project-worker-01 \
  --runtime nanobot \
  --project my-project \
  --role worker

# Edit workspace/ files (SOUL.md, AGENTS.md, USER.md)

bricklayer agent deploy --id my-project-worker-01
# → copies to ai-agents, commits, pushes
# → prints exact VPS commands to run

# SSH to VPS, run the printed commands
# Verify docker logs, then:
bricklayer agent live --id my-project-worker-01
```
